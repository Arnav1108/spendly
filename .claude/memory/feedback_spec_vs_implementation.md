description: When asked for a spec document, only write the spec — do not begin implementation

---

When the user asks for a spec document, write the spec and stop. Do not proceed to implementation.

**Why:** The user uses separate explicit commands for spec generation and implementation. Starting implementation after a spec request is unwanted and disruptive.

**How to apply:** Any time the user asks for a spec, plan, or design document — write it, report back, and wait. Only implement when the user gives a specific implementation command (e.g. via Plan Mode, a slash command, or an explicit "implement this" instruction).
